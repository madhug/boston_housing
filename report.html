Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer:

K-Fold cross-validation training technique is an iterative improvement on the train test split technique. With train test split, the data is split only once. So, we have a defined set of training data and test data based on how much of the data we are willing to set aside for testing. This means that we lose a certain amount of data. The model might miss learning from the remaining data set.

With K-Fold cross-validation, the train test split is done K times on the data set, each time picking a different subset of the data to be set aside as the test data set. The dataset is divided into K subsets of equal size. In each iteration, it is trained on K-1 datasets with the remaining 1 dataset used for validation. This means that the model is trained on the entire data set, while also being cross-validated. The final score is an average of the each iteration of training. This helps reduce noise in the score.

Using K-Fold to split the data for GridSearch algorithm provides it with K-1 training datasets and 1 testing dataset, which is cycled through the all the data. GridSearch algorithm generates a model for each combination of hyperparameters provided, and does K-Fold validation for all the models. K-Fold validation returns the average score for each of the models. GridSearch then returns the model with the best average score. This makes the model more robust.

In larger datasets, K-fold is used along with train-test split. Initially, a subset of data is set aside for testing. The remaining training dataset is then run through K-fold split, which does K iterations of the model fitting with cross-validation. This model is then tested on with initial test data that was set aside. This prevents "data leak" from occuring between the test, validation and training dataset. This prevents overfitting on validation data.

# Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV
from sklearn.tree import DecisionTreeRegressor

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    max_depth = [x for x in range(1, 11)]
    params = {'max_depth': max_depth}

    # Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # Create the grid search object
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
